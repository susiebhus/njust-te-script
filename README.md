# NJUST 教学评价一键填写脚本

一个用于南京理工大学本科教务系统教学评价页面的 Tampermonkey 用户脚本。脚本会在评教页面右下角生成一个浮动面板，支持一键选择评价选项，并提供保存、提交按钮的快捷入口。

> 本项目只做页面自动填写辅助。请在提交前自行检查每一项内容，确认评价符合个人真实意见。

## 功能特性

- 自动识别教学评价页面中的单选题组。
- 默认将每组评价选择为“很符合”。
- 自动填写页面中可识别的文字评价框。
- 在页面右下角生成浮动操作面板：
  - `一键很符合`
  - `保存`
  - `提交`
- 兼容页面已加载后再运行脚本的情况。
- 支持动态页面重新渲染后自动补回浮动面板。
- 不会自动提交，需要用户手动点击脚本面板或页面原生提交按钮。

## 适用页面

脚本匹配以下域名下的页面：

```text
*://bkjw.njust.edu.cn/*
```

主要面向本科教务系统中的学生评教页面，例如标题为“学生评教”的评价表单页面。

## 制作过程

### 1. 分析页面结构

根据实际评教页面截图和 Chrome 中打开的真实页面，评教表格大致由两部分组成：

- 左侧为评价指标。
- 右侧为每一行对应的一组选项：
  - 很符合
  - 较符合
  - 一般
  - 不符合
  - 极不符合

真实页面中，每个评价指标通常对应一组 `input[type="radio"]`，并通过相同的 `name` 属性组成一组。例如页面上可能有 12 组评价，每组 5 个选项，总计 60 个单选框。

### 2. 设计分组逻辑

脚本首先收集页面上所有可见、未禁用的单选框：

```js
document.querySelectorAll('input[type="radio"]')
```

然后按 `name` 属性进行分组。每一组单选框代表一条评价指标。

为了增强兼容性，脚本还处理了没有 `name` 属性的情况：如果某个单选框没有 `name`，脚本会尝试回退到它所在的表格行 `tr`，把同一行里的单选框视为一组。

这样可以适配一些老式教务系统页面或非标准表单结构。

### 3. 设置默认选择项

脚本使用配置项控制默认选择第几个选项：

```js
const CONFIG = {
    // 0 = 很符合，1 = 较符合，2 = 一般，3 = 不符合，4 = 极不符合
    defaultOptionIndex: 0,
};
```

当前默认值为 `0`，也就是每组选择第一个选项“很符合”。

选择时脚本不会只修改 `checked` 属性，而是会同时执行点击和事件触发：

```js
target.click();
["mousedown", "mouseup", "click", "input", "change"].forEach(type => trigger(radio, type));
```

这样做是为了兼容页面中可能存在的事件监听、总分计算或表单状态更新逻辑。

### 4. 自动填写文字评价

如果页面存在 `textarea`，或者存在名称、占位符、id 中带有“评价、意见、建议、备注”等含义的文本输入框，脚本会自动填入一段默认评价：

```js
老师教学认真负责，课程内容清晰，讲解准确，课堂安排合理，对学习很有帮助。
```

如果输入框设置了 `maxlength`，脚本会自动截断到允许长度，避免超过页面限制。

脚本不会填充普通文本框，避免误把课程编号、搜索框、隐藏字段等内容改掉。

### 5. 添加浮动操作面板

脚本运行后会在页面右下角插入一个浮动面板：

```text
一键很符合 | 保存 | 提交
```

其中：

- `一键很符合`：自动填写所有评价选项。
- `保存`：寻找页面原生“保存”按钮并点击。
- `提交`：寻找页面原生“提交”按钮并点击。

页面中的按钮文字可能带有空格，例如 `保 存`、`提 交`。脚本在匹配按钮时会移除空白字符，因此可以正常识别。

### 6. 处理加载时机

用户脚本可能在页面加载前执行，也可能在页面加载完成后才执行。脚本对此做了兼容：

```js
if (document.readyState === "loading") {
    window.addEventListener("load", () => {
        setTimeout(init, 1000);
    });
} else {
    setTimeout(init, 1000);
}
```

这保证了无论脚本什么时候被 Tampermonkey 注入，浮动面板都能正常出现。

同时，脚本使用 `MutationObserver` 监听页面变化，如果教务系统局部刷新导致按钮消失，也会自动尝试重新创建。

## 安装方法

### 1. 安装 Tampermonkey

先在浏览器中安装 Tampermonkey 扩展：

- Chrome / Edge：在扩展商店搜索 `Tampermonkey`
- Firefox：在附加组件商店搜索 `Tampermonkey`

### 2. 新建用户脚本

打开 Tampermonkey 管理面板，点击：

```text
添加新脚本
```

### 3. 复制脚本内容

将仓库中的 [main.js](./main.js) 内容完整复制到 Tampermonkey 编辑器中，保存即可。

### 4. 打开评教页面

进入南京理工大学本科教务系统，打开学生评教页面。如果脚本生效，页面右下角会出现操作面板。

## 使用方法

1. 打开需要评价的课程页面。
2. 等待页面加载完成，观察右下角是否出现脚本面板。
3. 点击 `一键很符合`。
4. 检查页面中的选项和文字评价是否符合你的真实意见。
5. 如需暂存，点击 `保存`。
6. 确认无误后，再点击 `提交`。

## 修改默认评价选项

如果不想默认选择“很符合”，可以修改 `main.js` 中的配置：

```js
const CONFIG = {
    // 0 = 很符合，1 = 较符合，2 = 一般，3 = 不符合，4 = 极不符合
    defaultOptionIndex: 0,
};
```

例如改成默认选择“较符合”：

```js
defaultOptionIndex: 1,
```

改成默认选择“一般”：

```js
defaultOptionIndex: 2,
```

## 修改默认文字评价

修改 `CONFIG.commentText` 即可：

```js
commentText: "老师教学认真负责，课程内容清晰，讲解准确，课堂安排合理，对学习很有帮助。",
```

建议填写符合自己真实感受的评价内容。

## 注意事项

- 脚本不会自动提交，但点击面板里的 `提交` 会触发页面原生提交按钮。
- 提交前请务必检查内容。
- 如果学校教务系统页面结构发生变化，脚本可能需要更新选择器或分组逻辑。
- 如果右下角没有出现按钮，可以尝试刷新页面，或确认 Tampermonkey 是否已启用该脚本。
- 本脚本仅供学习和个人便利使用。
